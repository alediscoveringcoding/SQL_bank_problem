# SQL_bank_problem
This project implements a relational database system for a credit institution that offers loans to both individuals and legal entities.
It includes full SQL scripts for table creation, data population, queries, and triggers to manage and track client information, loans, payments, and guarantees.

Features:

Five main tables: clienti_fizice, clienti_juridice, credite, plati, garantii.

Data integrity constraints (primary keys, foreign keys, and checks).

SQL queries for financial summaries, available resources, and client relationships.

Views and triggers for automated updates and data consistency.

Comprehensive schema for credit tracking, including payments and guarantees.

This database serves as a model for financial institutions managing clients, loans, and guarantees in an efficient and normalized relational structure.
